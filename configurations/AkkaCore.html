내부 Akka 시스템 설정

Play 2.0 요청 처리를 위해 내부적으로 Akka Actor 시스템을 이용하고 있습니다. Actor 시스템 설정은 application.conf 파일에서 할 수 있습니다.

Action invoker 배우

Action invoker 행위자는 Action 의 코드를 실행하는 데 사용됩니다. 여러 Action을 병렬로 실행하기 위해 여러 배우를 시작하고 그들을 Round Robin 라우터에서 제어하고 있습니다. 또한, 모든 배우들이 stateless.

Action invoker 행위자는 응답 본문을 해석하는 데 사용되는 바디 파서 를 취득하기 위해서도 사용됩니다. 이 검색 프로세스는 배우의 응답 (파싱에 사용되는 BodyParser 개체)를 대기하는 구현되어 있으며, 설정된 제한 시간을 경과하면 실패하는 사양입니다.

Action invoker 행위자는 actions-dispatcher 디스패처에 의해 실행됩니다.

Promise invoker 배우

Promise invoker 행위자는 Promise 의한 비동기 콜백을위한 배우입니다. 병렬 Promise 콜백을 실행하기 위해서는 그만큼 많은 Promise invoker 행위자가 사용 가능해야합니다. 또한 모든 Promise invoker 행위자는 상태입니다.

Promise invoker 행위자는 promises-dispatcher 디스패처에 의해 실행됩니다.

WebSockets agent 배우

각 WebSocket 연결 상태는 Agent 행위자에 의해 관리됩니다. WebSocket 연결할 때마다 새로운 Actor가 생성되고 소켓이 닫힌 때 Actor가 삭제됩니다. 이 행위자는 안정된입니다.

WebSocket agent 행위자는 websockets-dispatcher 디스패처에 의해 실행됩니다.

기본 설정

Play 2.0 사용하는 기본 설정을 다음과 같습니다. 응용 프로그램에서 덮어 않은 경우이 기준 설정이 사용됩니다. 필요에 따라 사용자 정의하십시오.

play {
    
    akka { 
        event - handlers =  [ "akka.event.slf4j.Slf4jEventHandler" ] 
        loglevel = WARNING
        
        actor {
            
            deployment {

                / actions {
                    router = round-robin
                    nr-of-instances = 24
                }

                / promises { 
                    router = round - robin
                    nr - of - instances =  24 
                }

            }
            
            retrieveBodyParserTimeout =  1 second
            
            actions - dispatcher =  { 
                fork - join - executor { 
                    parallelism - factor =  1.0 
                    parallelism - max =  24 
                } 
            }

            promises - dispatcher =  { 
                fork - join - executor { 
                    parallelism - factor =  1.0 
                    parallelism - max =  24 
                } 
            }

            websockets - dispatcher =  { 
                fork - join - executor { 
                    parallelism - factor =  1.0 
                    parallelism - max =  24 
                } 
            }

            default - dispatcher =  { 
                fork - join - executor { 
                    parallelism - factor =  1.0 
                    parallelism - max =  24 
                } 
            }
            
        }
        
    }
    
}
