로그 설정

Play 2.0 로깅 구현은 logback 가 사용되고 있습니다.

application.conf에서 로그 수준 설정

로그 수준을 설정하는 가장 쉬운 방법은 conf / application.conf 의 logger 는 키를 사용하는 것입니다.

Play는 기본적으로 application 이라는 로거가 준비되어 있고, 기본 Logger 를 이용하여 로그를 출력하려면이 사용됩니다.

# Root logger : 
logger = ERROR

# Logger used by the framework : 
logger . play = INFO

# Logger provided to your application : 
logger . application = DEBUG

가장 상위 계층의 logger 설정은 모든 로그 출력에 영향을줍니다. 또한 특정 라이브러리에 대한 로그 수준 지정도 application.conf에 설명합니다. 예를 들어, Spring에 TRACE 로그 수준을 설정하려면 다음과 같이 설명합니다.

logger . org . springframework = TRACE

logback 설정

기본적으로 2 개의 appender가 정의되어 있습니다. 하나는 표준 출력에 로그, 또 하나는 logs / application.log 에 출력하는 로그입니다.

logback을 완전히 사용자 정의 할 수도, 설정 파일 conf / logger.xml 를 정의하기 만하면됩니다. Play가 사용 기본 설정 파일은 다음과 같습니다.

<configuration>
    
  <conversionRule  conversionWord = "coloredLevel"  converterClass = "play.api.Logger $ ColoredLevel"  />
  
  <Appender  name = "FILE"  class = "ch.qos.logback.core.FileAppender" > 
     <file> $ {application.home} / logs / application.log </ file> 
     <encoder> 
       <pattern> % date - % level - from % logger in % thread % n % message % n % xException % n </ pattern> 
     </ encoder> 
   </ appender>

  <Appender  name = "STDOUT"  class = "ch.qos.logback.core.ConsoleAppender" > 
    <encoder> 
      <pattern> % coloredLevel % logger {15} - % message % n % xException {5} </ pattern> 
    </ encoder> 
  </ appender>
  
  <Logger  name = "play"  level = "INFO"  /> 
  <Logger  name = "application"  level = "INFO"  />

  <루트  level = "ERROR" > 
    <appender-ref  ref = "STDOUT"  /> 
    <appender-ref  ref = "FILE"  /> 
  </ root>
  
</ configuration>

logback 설정 파일의 변경

또한 시스템 속성에 의해 다른 logback 설정 파일을 지정할 수 있습니다.
프로덕션 (production) 모드에서 실행하는 경우에 매우 유용합니다.

-Dlogger.resource 을 사용

다른 logback 설정 파일을 클래스 패스로부터로드하도록 지정합니다 :

$ start - Dlogger . resource = conf / prod - logger . xml

-Dlogger.file 을 사용

다른 logback 설정 파일을 파일 시스템에서로드하도록 지정합니다.

$ start - Dlogger . file = / opt / prod / logger . xml

-Dlogger.url 을 사용

다른 loback 설정 파일을 URL에서로드하도록 지정합니다 :

$ start - Dlogger . url = http : / / conf.mycompany.com / logger.xml
