좋아하는 IDE에서 개발하는

Play에서 개발하는 것은 간단합니다. Play는 자동으로 소스 파일에 변경 내용을 컴파일하고 변경 내용을 재 갱신하기 때문에 정교한 IDE 필요없이 간단한 텍스트 편집기로 쉽게 개발할 수 있습니다.

그러나 현대적인 Java, Scala IDE는 자동 컴파일, 실행중인 컴파일, 리팩토링 어시스트, 디버깅과 같은 멋진 생산적인 기능을 제공하고 있습니다.

Eclipse

설정 파일의 생성

Play 간단한 Eclipse 설정을위한 명령을 제공합니다. Play 응용 프로그램을 Eclipse 프로젝트로 변환하려면 eclipsify 명령을 사용합니다.

소스 jar 파일이 필요없는 경우에는 다음 명령을 실행합니다.

[ My first Application ] $ eclipsify

사용 가능한 소스 jar 파일을 검색하려면 다음 명령을 실행합니다. (이 경우에는 더 많은 시간이 걸리고 몇 가지 소스가 아닌되어 있을지도 모릅니다.)

[ My first Application ] $ eclipsify with - source = true

다음 File / Import / General / Existing project ... 메뉴를 사용하여 작업 영역에 응용 프로그램을 가져올 수 있습니다. (처음에 당신의 프로젝트를 컴파일합니다.)

play debug run 에서 응용 프로그램을 시작하면 Debug AS 를 사용하여 디버깅 세션을 시작할 수 the Connect JPDA launcher를 사용할 수 있습니다.
디버깅 세션을 중지하면 서버가 중지되지 않습니다.

Tip : ~ run 을 사용하면 파일의 변경시 직접 컴파일을 사용하여 응용 프로그램을 실행할 수 있습니다. 그러면 view 에 새로운 템플릿 파일이 생성 된 것을 자동으로 감지하고 파일에 변화가 생길 때 자동으로 컴파일됩니다. 보통 run 명령을 사용하고 있으면 매번 브라우저에서 새로 고침을하지 않으면 안됩니다.

클래스 경로 변경과 같은 응용 프로그램에서 중요한 변경 한 경우 eclipsify 명령을 사용하여 설정 파일을 다시 작성합니다.

Tip : 팀에서 작업을 수행하는 경우 Eclipse의 설정 파일은 커밋하지 않도록하십시오.

생성 된 설정 파일은 당신의 프레임 워크 설치에 대한 절대 참조를 포함합니다. 이것은 당신의 자신의 설치에 관한 설정입니다. 당신이 팀에서 작업 할 때 각 개발자는 Eclipse의 설정 파일을 개인에 둘 필요가 있습니다.

IntelliJ

설정 파일의 생성

Play 단순한 Intellij IDEA 설정을위한 명령을 제공합니다. Play 응용 프로그램을 IDEA 모듈로 변환하려면 idea 명령을 사용합니다.

소스 jar 파일이 필요없는 경우에는 다음 명령을 실행합니다.

[ My first Application ] $ idea

사용 가능한 소스 jar 파일을 검색하려면 다음 명령을 실행합니다. (이 경우에는 더 많은 시간이 걸리고 몇 가지 소스가 아닌되어 있을지도 모릅니다.)

[ My first Application ] $ idea with - sources

그 프로젝트에 응용 프로그램을 가져올 수 있습니다. (File-> New Module-> Import existing Module)

Tip : Scala 플러그인이 설치되어있는 경우, Java 기반 Play2 어플에서 IntelliJ의 문제 가 발생하고 있습니다. 이 문제가 해결 될 때까지, Scala 플러그인을 비활성화하는 것이 좋습니다됩니다.

디버깅을 수행하기 위해 먼저 디버깅 설정을 추가합니다.

  • Run / Debug Configurations 설정 대화 상자를 열고 Run -> Edit Configurations을 클릭합니다.
  • Remote 설정을 추가하고 Remote 를 선택합니다.
  • 다음 내용을 설정합니다.
    • 이름을 설정한다.
    • Transport : Socket을 설정한다.
    • Debugger mode : Attach를 설정한다.
    • Host : localhost를 설정한다.
    • Port : 9999을 설정한다.
    • 가져온 모듈을 선택한다.
  • Apply를 클릭하여 대화 상자를 닫습니다.

play를 디버그 모드에서 시작합니다.

$ play debug

다음과 같은 출력이됩니다.

Listening  for transport dt_socket at address :  9999

일부 중단 점을 설정합니다. play 작업을 수행하고 응용 프로그램을 시작합니다. ( play debug 를 실행 한 터미널에서도 동일한 일을 다시 실행해야합니다.) http://localhost:9000 을 검색하면 IntelliJ 중단 점에서 처리가 멈추어있는 것입니다.

클래스 경로 변경과 같은 응용 프로그램에서 중요한 변경 한 경우, idea 명령을 사용하여 설정 파일을 다시 작성합니다.

Netbeans

설정 파일의 생성

Play는 현재 기본 Netbeans 프로젝트의 생성을 지원하지 않습니다. 지금은 NetBeans SBT Plugin 을 사용 NetBeans의 Scala 프로젝트를 생성 할 수 있습니다.

먼저 plugins.sbt 파일을 편집합니다.

resolvers + =  { 
  "remeniuk repo" at "http://remeniuk.github.com/maven"  
}

libraryDependencies + =  { 
  "org.netbeans"  % %  "sbt-netbeans-plugin"  %  "0.1.4" 
}

편집이 완료되면 다음 명령을 실행합니다.

$ play netbeans
