CoffeeScript를 사용

CoffeeScript 는 작고 우아한 언어로, JavaScript에 컴파일됩니다. CoffeeScript를 사용하면 JavaScript 코드를 더 나은 구문으로 쓸 수 있습니다.

Play 2.0 CoffeeScript 등 다른 언어로 컴파일 된 같은 자산은 모든 app / assets 에 넣게되어 있습니다. 여기에 담긴 파일은 빌드에서 자동으로 컴파일됩니다. CoffeeScript의 경우,이 때 보통 JavaScript에 컴파일됩니다. 생성 된 JavaScript 파일은 public / 디렉토리에 배치 된 것처럼 처리되므로 일단 컴파일되어 버리면 보통 JavaScript 파일과 차이는 없습니다.

참고 : 생성 된 JavaScript 파일 등의 Play가 관리하는 자원 자체는 public / 디렉토리에 직접 복사되는 것이 아니라, target/scala-2.9.1/resources_managed 라는 다른 디렉토리에 저장됩니다.

예를 들어, app / assets / javascripts / main.coffee 는 public / javascripts / main.js 에서 통상의 JavaScript 리소스로 사용할 수 있습니다.

CoffeeScript 소스 파일은 compile 명령을 실행할 때와 개발 모드에서 응용 프로그램을 시작하는 동안 브라우저에서 페이지를 다시로드 될 때 자동으로 컴파일됩니다. 후자의 타이밍에 컴파일 된 경우 컴파일 오류는 브라우저에 표시됩니다.

디렉토리 구조

CoffeeScript을 사용하는 프로젝트의 기본 데이레쿠토리 구조는 다음과 같습니다.

app
  └ Assets
     └ 자바 스크립트 나
        └ Main . coffee

두 JavaScript 파일 : public / javascripts / main.js 와 public / javascripts / main.min.js 가 컴파일됩니다. 하나 번째 파일은 개발시 유용한 읽을 수있는 코드로, 둘째는 프로덕션 환경에서 사용할 수있는 압축 파일입니다. 모두 템플릿에서 사용할 수 있습니다 :

<script  src = "@ routes.Assets.at (" javascripts / main . js ")" >
<script  src = "@ routes.Assets.at (" javascripts / main . min . js ")" >

옵션

CoffeeScript 컴파일은 ( PlayProject 설정 부분이다) Build.scala 파일에서 제어 할 수 있습니다. 현재 지원되는 유일한 옵션은 bare 모드입니다.

coffeescriptOptions : =  Seq ( "bare" )

기본 CoffeeScript 컴파일을 사용 실험적인 옵션이 있음을 기억하십시오. 이 방법의 장점은 속도이며, 단점은 외부에 의존합니다. 이 방법을 시도하려면 설정에 다음을 추가하십시오 :

coffeescriptOptions : =  Seq ( "native" ,  "/ usr / local / bin / coffe-p" )

글로벌 범위의 오염을 방지하기 위해 JavaScript 코드는 기본적으로 최상위의 안전한 함수 나팔에 생성됩니다. bare 옵션이 함수 래퍼를 제거합니다.
