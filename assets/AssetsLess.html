LESS CSS를 사용

LESS CSS 는 동적 스타일 시트를 작성하기위한 언어입니다. LESS CSS를 사용하면 변수와 믹스 인 등을 활용 한 매우 유연한 CSS를 작성할 수 있습니다.

Play는 다른 언어로 컴파일 된 자산은 app / assets 에 넣습니다. LESS CSS를 여기에 넣어두면, 빌드에서 보통 CSS 파일에 컴파일됩니다. 생성 된 CSS는 public / 디렉토리에 배치 된 것처럼 처리되므로 일단 컴파일되어 버리면 보통의 CSS 파일과 차이는 없습니다.

참고 : 생성 된 CSS 파일 등의 Play가 관리하는 자원 자체는 public / 디렉토리에 직접 복사되는 것이 아니라, target/scala-2.9.1/resources_managed 라는 다른 디렉토리에 저장됩니다.

예를 들어, app / assets / stylesheets / main.less 의 LESS 소스 파일은 결국 public / stylesheets / main.css 됩니다.

LESS 소스 파일은 compile 명령 실행하거나, 개발 모드에서 브라우저에서 페이지를 다시로드 할 때 자동으로 컴파일됩니다. 컴파일 오류는 브라우저에 출력됩니다.

LESS 코드 분할

LESS의 import 기능을 이용하면 LESS 코드를 여러 라이브러리에 분할 할 수 있습니다.

라이브러리가 개별적으로 컴파일 및 import되어 버리는 일이 없도록 분할 된 LESS 파일은 컴파일러에 처리하지 않도록해야합니다. 이를 위해 분할 된 LESS 파일 이름에는 밑줄 ( _ )을 앞에 두어되어 있습니다. 예를 들어, _mylibrary.less 는 같은 파일 이름을 지정합니다. 이 동작을 제어하려면이 페이지의 마지막에있는 설정 단락을 참조하십시오.

디렉토리 구조

LESS를 이용할 때의 디렉토리 구조는 다음과 같습니다.

app
  └ Assets
     └ StyleSheets
        └ Main . less
        └ utils
           └ _reset . less
           └ _layout . less

다음과 같은 내용의 main.less 가 있다고합니다.

@ import  "utils / _reset.less" ; 
@ import  "utils / _layout.less" ;

h1 { 
    color : red ; 
}

이 LESS는 public / stylesheets / main.css 라는 CSS 파일에 컴파일 된 일반 공개 자산과 마찬가지로 템플릿에서 호출 할 수 있습니다. 축소판도 같이 생성됩니다.

<link  REL = "stylesheet"  href = "@ routes.Assets.at (" stylesheets / main . css ")" >
<link  REL = "stylesheet"  href = "@ routes.Assets.at (" stylesheets / main . min . css ")" >

설정

밑줄이 추가되지 않은 모든 파일을 컴파일하는 기본 동작, 예를 들면, 그렇게 설계되지 않은 라이브러리가 포함되어있는 경우와 같이, 모든 프로젝트에 적합하지 않을지도 모릅니다 .

이 동작은 project / Build.scala 에서 lessEntryPoints 키를 덮어 써서 제어 할 수 있습니다. 이 키는 PathFinder 을 보유하고 있습니다.

예를 들어, app / assets / stylesheets / main.less 이외에는 아무것도 컴파일하지 않으면 다음과 같이합니다.

val main =  PlayProject ( appName , appVersion , mainLang = SCALA ). settings ( 
   lessEntryPoints << = baseDirectory ( _ /  "app"  /  "assets"  /  "stylesheets"  **  "main.less" ) 
 )
