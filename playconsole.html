Play 2.0 콘솔을 사용

콘솔 시작

Play 2.0 콘솔은 sbt를 기반으로 한 개발 콘솔입니다. Play 응용 프로그램의 모든 개발주기를 지원 해줍니다.

콘솔을 시작하려면, Play 응용 프로그램 디렉토리에서 play 스크립트를 실행하십시오.

$ cd / path / to / any / application
$ play

도움말보기

help play 명령을 실행하면 사용 가능한 명령과 간단한 설명을 볼 수 있습니다.

[ My first Application ] $ help play

개발 모드로 서버를 시작

응용 프로그램을 개발 모드로 부팅하기 위해서는, run 명령을 이용합시다.

[ My first Application ] $ run

이 모드에서 서버는 핫 전개 기능이 활성화 된 상태에서 서버를 시작합니다. 즉, 요청이 올 때마다 Play 전체 프로젝트를 확인하고 필요한 소스 코드 만 다시 컴파일합니다. 서버를 다시 시작해야하는 경우도 자동으로 수행합니다.

컴파일 오류가 발생했을 때 그 내용을 브라우저에서 직접 확인할 수 있습니다.

서버를 정지 시키려면, Ctrl + D 키를 입력합니다. 서버가 중지하고 Play 콘솔 프롬프트로 돌아갑니다.

컴파일

Play 2.0 compile 명령을 사용하여 서버를 시작하지 않고 응용 프로그램을 컴파일 할 수 있습니다.

[ My first Application ] $ compile

대화 콘솔 시작

console 명령을 실행하면 Scala의 REPL이 시작하여 응용 프로그램 코드를 대화식으로 테스트 할 수 있습니다.

[ My first Application ] $ console

디버그 모드

JPDA (Java Platform Debugger Architecture)의 디버그 포트의 사용을 콘솔을 시작할 때 지정할 수 있습니다. 디버그 모드의 사용에는 play debug 명령을 사용합니다.

$ play debug

JPDA 포트를 사용할 수있게되면, JVM은 부팅시 다음과 같은 로그를 출력합니다.

Listening  for transport dt_socket at address :  9999

Note : play debug 이용하면 기본적으로 JPDA 소켓은 9999 포트에서 열립니다. set JPDA_PORT = 1234 과 같이 환경 변수 JPDA_PORT 에서 포트를 구성 할 수 있습니다.

sbt 기능을 이용

Play 콘솔은 보통 sbt 콘솔이기도하므로, triggered execution 같은 sbt 기능도 이용할 수 있습니다.

예 ~ compile 명령을 실행하면 소스 코드를 변경할 때마다 필요한 소스 코드 만 컴파일 할 수 있습니다.

[ My first Application ] $ ~ compile

컴파일 소스 파일을 변경할 때마다 실행됩니다.

만약 ~ run 명령을 실행하는 경우

[ My first Application ] $ ~ run

개발 서버의 가동 중은 컴파일 실행은 활성화됩니다.

마찬가지로 ~ test 명령을 실행하면 소스 코드가 변경 될 때마다 테스트가 실행됩니다.

[ My first Application ] $ ~ test

Play 명령을 직접 실행

Play 콘솔을 시작하지 않고 명령을 직접 실행할 수 있습니다. 예를 들어, play run 은 다음과 같이 시작할 수 있습니다.

$ play run
 [ info ]  Loading Project Definition from myFirstApp / project
 [ info ]  Set Current Project to My first Application

---  ( Running the Application from SBT ,  auto - reloading is enabled )  ---

[ info ] play -  Listening  for HTTP on Port 9000. ..

( Server started ,  use  Ctrl + D to stop and go Back to the console ...)

응용 프로그램을 직접 부팅 할 때 서버 정지를위한 Ctrl + D 명령을 사용하면 즉시 OS의 프롬프트로 돌아 가기 것이다.

강제 청소

뭔가 잘못 sbt 캐시가 방해 된 경우, 모든 생성 된 디렉토리를 청소하기 위해 OS의 명령 줄 도구 clean-all 명령을 실행합니다.

$ play clean - all
