Play 2.0 소스 코드에서 빌드

초기 베타 릴리스의 최신 개선 및 버그 수정의 혜택을 받고 싶은 경우, Play 2.0 소스 코드에서 컴파일합시다. 소스 코드는 Git client 에서 가져옵니다.

$ git clone git : / / github.com/playframework/Play20.git

소스 코드를 얻으면 Play20/framework 디렉토리로 이동하여 build 스크립트를 시작하고 sbt의 빌드 콘솔로 이동합니다.

$ CD Play20 / framework
$ . / build
 > build - repository

sbt 콘솔에있을 때, build-repository 작업을 수행하여 전체 프레임 워크를 컴파일하자. 이 작업은 Play가 의존하는 모든 종속 모듈을 포함 로컬 Ivy 저장소를 만듭니다.

sbt를 별도로 설치할 필요가 없습니다. Play 2.0에 필요한 버전의 sbt (현재는 sbt 0.11.2)이 이미 포함되어 있기 때문입니다.

만약 Play 자체를 변경 한 경우는 compile 과 publish-local 을 사용하여 프레임 워크를 다시 빌드하는 것이 좋습니다.

테스트를 실행하는

sbt 콘솔 test 작업을 시작하면 기본적인 테스트를 수행 할 수 있습니다.

> test

우리는 샘플 애플리케이션을 통해 프레임 워크의 테스트도 실시하고 있습니다. runtests 스크립트를 실행하면 샘플에 대한 테스트를 포함한 전체 테스트 스위트를 실행 할 수 있습니다.

$ . / runtests

프로젝트 만들기

소스 코드에서 직접 구축 한 버전의 Play를 사용하여 프로젝트를 만드는 방법은 일반 Play 응용 프로그램을 만드는 방법과 거의 동일합니다.

export PATH = $ PATH : / Play20

이미 Play 2.0 Beta에서 개발 된 Play 2.0 응용 프로그램을 직접 빌드 한 버전으로 업그레이드 할 때,

resolvers + + =  Seq ( 
  ... 
  Resolver . file ( "Local Repository" , file ( "<projdir> / Play20/repository/local" )) ( Resolver . ivyStylePatterns ), 
  ... 
)

addSbtPlugin ( "play"  %  "sbt-plugin"  %  "2.1-SNAPSHOT" )

같은 설정을 project / plugins.sbt에 추가합시다.

코드를 eclipse에서 열

Play 코드를 eclipse에서 여는 데 필요한 설정은 Stackoverflow 를 참조하십시오.
