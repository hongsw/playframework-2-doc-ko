API 디자인

  • Play는 Java, Scala위한 프레임 워크입니다. 변경된 API가 Java, Scala 모두에서 동작하도록합시다.
  • Java API는 framework / play / src / main / java 디렉토리에 배치하고 패키지 구성은 play.myapipackage.xxxx 합시다.
  • Scala API는 framework / play / src / main / scala 디렉토리에 배치하고 패키지 구성은 play.api.myapipackage 합시다.
  • Java 및 Scala API는 다음과 같이 구현합시다.
  • 코어 API를 Scala로 구현합니다 ( play.api.xxx )
  • API의 내부 구성 요소 라이프 사이클 관리가 필요하거나 대체 가능한 필요가있는 경우는 플러그인으로 만들어 보자. 해당하지 않는 경우,이 단계를 건너 뜁니다.
  • 코어 API를 Scala 사용자를 위해 랩합니다 ( 샘플 )
  • Scala API를 Java 사용자를 위해 랩합니다 ( 샘플 )
  • 기능 추가에 제한이 없습니다. 그 새로운 기능은 프레임 워크의 핵심에 필요한 플러그인으로 구현해야 하는가라는 것을 생각합시다.
  • 의문점이 있으면,에 물어 봅시다.

테스트 및 문서

  • 모든 공개 메소드, 공개 클래스는 사용 예제 및 설명 등 scaladoc와 javadoc를 작성합시다.
  • 모든 기능이나 버그 수정에 대한 기능적 테스트 ( framework / integrationtest )과 spec ( / play / src / test )을 만들어 보자.
  • push 전에 Play 결합 테스트 스위트 framework / runtests 을 실행합시다. 테스트가 실패하면 무시하지 않고 문제를 해결합시다.

코드 스타일

  • 커밋 전에 scalariform-format을 실행합시다.

git 커밋

  • 적극적으로 rebase합시다.
  • 동일한 주제에 대한 변경은 하나의 커밋으로합시다.
