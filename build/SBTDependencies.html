라이브러리 의존성 관리

관리되지 않는 의존성

대부분의 개발자는 결국 "관리 된 의존성"을 이용하게되지만, 처음에는 "관리되지 않는 의존성"에서 시작하는 것이 알기 쉬운 수 있습니다.

"관리되지 않는 의존성"은 다음과 같이 사용합니다 : 프로젝트 루트에 lib / 디렉토리를 만들고이 디렉토리에 jar 파일을 추가합니다. 이들은 자동으로 응용 프로그램의 클래스 경로에 추가됩니다. 그거야!

이 경우 project / Build.scala 에 의존성을 기술 할 필요도 없습니다. 그러나 lib / 외부에 JAR 파일을 저장할 경우, project / Build.scala 그런 설정을 넣어도 좋습니다.

관리 된 의존성

Play 2.0은 "관리 된 의존성"을 실현하기 위해 Apache Ivy를 이용하고 있습니다. 만약 Maven이나 Ivy에 詳しけれ 경우 특히 곤란한 것은 아닐까요.

대부분의 경우, 종속 모듈을 project / Build.scala 에 나열하는 것만으로 좋다. 필요한 경우, Maven의 POM 파일과 Ivy 설정 파일에 종속 모듈을 쓰고, 그것을 sbt에서 읽는 수 있습니다.

의존성의 정의는 다음과 같이 쓰고 있습니다. ( group , artifact , revision 을 정의합니다.)

val appDependencies =  Seq ( 
  "org.apache.derby"  %  "derby"  %  "10.4.1.3" 
)

또는 선택적으로 지정할 수 configuration 을 첨가 할 수 있습니다.

val appDependencies =  Seq ( 
  "org.apache.derby"  %  "derby"  %  "10.4.1.3"  %  "test" 
)

sbt (내부 Ivy를 사용합니다)은 모듈의 다운로드 원을 알고 있어야합니다. 모듈이 sbt에 내장 된 기본 저장소 중 하나에 포함되어 있으면, 특히 아무것도하지 않고 움직입니다.

% % 를 사용하여 적절한 Scala 버전을 선택

groupID % artifactID % revision 대신, groupID % % artifactID % revision 라고하는 것처럼, groupID로부터 2 % 를 두 겹쳐 써서, sbt는 프로젝트에 설정된 Scala 버전에 대응 한 아티팩트를 다운로드 줍니다.

그냥 단축키이며, % % 를 사용하지 않고 쓸 수 있습니다.

val appDependencies =  Seq ( 
  "org.scala-tools"  %  "scala-stm_2.9.1"  %  "0.3" 
)

예를 들어, 빌드 파일 scalaVersion 에 2.9.1 을 지정하는 경우, 다음의 정의도 완전히 같은 의미입니다.

val appDependencies =  Seq ( 
  "org.scala-tools"  % %  "scala-stm"  %  "0.3" 
)

확인자

모든 패키지가 같은 서버에있는 것은 아닙니다. sbt 표준 Maven2 저장소와 Scala Tools Releases 리포지터리 ( http://scala-tools.org/repo-releases )를 기본으로 사용합니다. 정의한 의존 모듈이 기본 저장소에없는 경우, Ivy가 모듈을 찾을 수 있도록, 확인자 직접 추가 해 줄 필요가 있습니다.

resolvers 라는 설정 키를 사용하여 자신의 확인자을 추가합니다.

resolvers + = name at location

예를 들어,

resolvers + =  "Scala-Tools Maven2 Snapshots Repository" at "http://scala-tools.org/repo-snapshots"

로컬 Maven 저장소를 만든 다음, sbt에 검색 할 수 있습니다.

resolvers + =  ( 
    "Local Maven Repository" at "file :/ /" + Path . userHome . absolutePath + "/ .m2/repository" 
)

정리

비표준 확인자와 일부 관리 된 의존성을 포함한 프로젝트의 정의를 보여줍니다.

import SBT . _
 import  Keys . _
 import  PlayProject . _

object  ApplicationBuild  extends  Build  {

    val appName          =  "My first application" 
    val appVersion       =  "1.0"

    val appDependencies =  Seq (
        
      "org.scala-tools"  % %  "scala-stm"  %  "0.3" , 
      "org.apache.derby"  %  "derby"  %  "10.4.1.3"  %  "test"
      
    )

    val main =  PlayProject ( appName , appVersion , appDependencies ). settings ( defaultScalaSettings : _ *). settings (
      
      resolvers + =  "JBoss repository" at "https://repository.jboss.org/nexus/content/repositories/" , 
      resolvers + =  "Scala-Tools Maven2 Snapshots Repository" at "http://scala-tools.org/ repo-snapshots "
            
    )

}
