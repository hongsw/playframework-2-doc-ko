빌드 시스템

Play 2.0 빌드 시스템은 Scala 및 Java 프로젝트에 대한 최소한으로 강요하는 듯해 않은 빌드 도구 인 sbt 을 기반으로합니다.

/ project 디렉토리

모든 빌드 설정 project 디렉토리에 저장됩니다. 이 폴더에는 3 가지 주요 파일이 포함되어 있습니다 :

  • build.properties : 사용 sbt 버전을 설명하는 지표 인 파일입니다.
  • Build.scala : 응용 프로그램 프로젝트 빌드 설명자입니다.
  • plugins.sbt : 프로젝트 빌드에 사용되는 SBT 플러그인입니다.

Play 2.0 응용 프로그램의 기본 빌드

play new 명령에 의해 생성 된 기본 빌드이 있습니다 :

import SBT . _
 import  Keys . _
 import  PlayProject . _

object  ApplicationBuild  extends  Build  {

  val appName          =  "Your application" 
  val appVersion       =  "1.0"

  val appDependencies =  Seq ( 
    / / Add your project dependencies here, 
  )

  val main =  PlayProject ( 
    appName , appVersion , appDependencies , mainLang = SCALA
   ). settings ( 
    / / Add your own project settings here       
  )

}

응용 프로그램 이름, 버전 및 의존성 등의 표준 옵션을 쉽게 정의 할 수 있도록하기 위해이 같은 방식으로 작성되어 있습니다.

sbt의 모든 기능을 Play 2.0 프로젝트에서 사용할 수있는 점에 유의하십시오.

sbt 용 Play 플러그인

Play 콘솔이나 동적 다시로드 등의 개발 기능은 모든 sbt 플러그인으로 구현되어 있습니다. 이것은 plugins.sbt 파일에 등록되어 있습니다 :

addSbtPlugin ( "play"  %  "sbt-plugin"  %  "2.0" )

Typesafe 저장소 확인자에 추가해야 할 수도 있습니다. http://github.com/playframework/Play20/wiki/Repositories 를 참조하십시오.
