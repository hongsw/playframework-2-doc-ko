추가 설정

응용 프로그램을 프로덕션 모드에서 실행할 때 모든 설정을 덮어 쓸 수 있습니다. 이 섹션에서는이 "추가 설정"일반적인 사례를 소개합니다.

추가 설정은 Java 시스템 속성을 사용하여 지정합니다. Play 의해 생성 된 start 스크립트를 사용하는 경우, Java 시스템 등록 정보 테이을 start 스크립트에 직접 지정할 수 있습니다.

HTTP 서버의 주소와 포트를 지정

HTTP 포트와 주소를 모두 지정할 수 있습니다. 기본 포트는 9000 , 주소 0.0.0.0 (모든 주소)입니다.

$ start - Dhttp . port = 1234  - Dhttp . address = 127.0 . 0.1

이 설정은 기본 내장 Netty 서버 만 이용합니다.

추가 JVM의 시작 옵션

start 스크립트에 JVM의 시작 옵션을 지정할 수 있습니다. 지정하지 않은 경우는, JVM의 기본 설정이 사용됩니다.

$ start - Xms128M  - Xmx512m  - server

대체 설정 파일을 지정하는

기본적으로로드되는 설정 파일은 클래스 패스에 application.conf 입니다. 필요에 따라 다른 설정 파일을 지정할 수 있습니다.

-Dconfig.resource 을 사용

응용 프로그램의 클래스 경로에서 대체 구성 파일을 검색합니다. (일반적으로 이러한 대체 설정 파일을 패키징 전에 응용 프로그램의 conf / 디렉토리에 넣어두면 좋을 것입니다.)

$ start - Dconfig . resource = prod . conf

-Dconfig.file 을 사용

응용 프로그램의 바이너리에 패키징되어 있지 않은 파일 시스템의 설정 파일을 지정할 수 있습니다.

$ start - Dconfig . file = / opt / conf / prod . conf

-Dconfig.url 을 사용

모든 URL을 지정하여 설정 파일을로드 할 수 있습니다.

$ start - Dconfig . url = http : / / conf.mycompany.com / conf / prod.conf

이 때, prod.conf 에 다음과 같이 include 명령을 작성하면 원래 설정 파일을 볼 수 있습니다.

include "application.conf"

key . to . override = blah

특정 설정 키를 덮어

설정 파일 전체를 대체 할 것이 아니라 일부 얹어 값 키를 덮어 있으면 좋은 것 또한있을 것이다. 이 경우 Java 시스템 속성을 이용하여 다음과 같이합니다.

$ start - Dapplication . secret = verysecretkey - Ddb . default . password = toto

환경 변수를 사용

application.conf 에서 환경 변수를 참조 할 수 있습니다.

my . key = DefaultValue
 my . key = $ {? MY_KEY_ENV }

필드를 덮어 my.key = $ {? MY_KEY_ENV} 라는 설명을했습니다. 이것은 MY_KEY_ENV 라는 값이없는 경우는 그냥 없었던 일로됩니다. 그러나 MY_KEY_ENV 라는 환경 변수가 설정되어있는 경우, 환경 변수의 것이 사용됩니다.

logback 설정 파일을 수정

시스템 속성을 사용하여 다른 logback 설정 파일을 지정할 수 있습니다.

-Dlogger.resource 을 사용

다른 logback 설정 파일을 클래스 패스로부터로드 할 수 있습니다.

$ start - Dlogger . resource = conf / prod - logger . xml

-Dlogger.file 을 사용

다른 logback 설정 파일을 파일 시스템에서 읽을 수 있습니다.

$ start - Dlogger . file = / opt / prod / logger . xml

-Dlogger.url 을 사용

다른 logback 설정 파일을 URL에서 가져올 수 있습니다.

$ start - Dlogger . url = http : / / conf.mycompany.com / logger.xml
