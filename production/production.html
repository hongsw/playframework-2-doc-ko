응용 프로그램을 프로덕션 모드에서 시작

Play 어플리케이션을 실전 모드로 부팅하는 방법은 여러 가지가 있습니다. 우선 로컬 환경에 Play를 설치하는 가장 간단한 방법을 시도해 보자.

start 명령을 사용

어플리케이션을 실전 모드로 부팅하는 가장 쉬운 방법은 Play 콘솔에서 start 명령을 실행하는 것입니다. 그래서 서버에 미리 Play 2.0을 설치해야합니다.

[ My first Application ] $ start

run 명령은 개발 모드 위해서만 된 것이므로, 운영 환경에서는 절대로 사용하지 마십시오. 개발 모드 요청마다 sbt가 모든 소스 파일의 업데이트 확인을 가버립니다.

start 명령을 실행하면 Play는 JVM을 새로운 포크하여 표준 Netty HTTP 서버를 시작합니다. 그 때, 표준 출력 Play 콘솔로 리디렉션되므로 거기에서 응용 프로그램의 상태를 모니터링 할 수 있습니다.

서버 시작시 서버의 프로세스 ID가 나타날 때, RUNNING_PID 라는 파일에 기록됩니다. 실행중인 Play 서버를 중지시키기 위해, 해당 프로세스에 SIGTERM 신호를 전송하여 응용 프로그램을 종료하면 좋을 것입니다.

Ctrl + D 를 입력하면 Play 콘솔은 종료되지만, 생성 된 서버 프로세스는 백그라운드에서 계속 실행됩니다. 그 때, 포크 한 JVM의 표준 출력은 닫혀 버립니다 만, 로그는 logs / application.log 라는 파일에 출력 계속됩니다.

Ctrl + C 를 입력하면 Play 콘솔과 포크 한 Play 서버 모두 JVM을 종료합니다.

위의 방법 외에, OS의 명령 프롬프트에서 play start 를 직접 실행해도 같은 수 있습니다.

$ play start

Note :-Dhttp.port 시스템 변수를 전달하여 HTTP 포트 번호를 변경할 수 있습니다.

stage 작업을 사용

start 명령의 문제점은 응용 프로그램을 대화식으로 시작하는 수작업이 필요하다는 것과, 프로세스를 분리하려면 Ctrl + D 가 필요하다는 것입니다. 즉,이 방법은 배포 자동화에 적합하지 않습니다.

이런 때 stage 작업을 사용하면 응용 프로그램이 나중에 즉시 시작할 수 있도록 준비를 할 수 있습니다. 프로젝트를 나중에 즉시 실행할 수 있도록하는 전형적인 명령은 다음과 같다.

$ play clean compile stage

이 명령은 응용 프로그램을 컴파일 할 때 생성 된 임시 파일이 제거되고 다시 컴파일이 실행되고 필요한 종속 모듈을 다운로드, 이들 모두가 target / staged 디렉토리에 복사됩니다. 또한 Play 서버를 시작하는 target / start 라는 스크립트도 생성됩니다.

응용 프로그램은 다음과 같이 시작할 수 있습니다.

$ target / start

생성 된 start 스크립트는 아주 간단합니다. 실제로,이 스크립트를 사용하지 않고 java 명령은 직접 응용 프로그램을 시작해도 좋을 정도입니다.

만약 Play가 서버에 설치되어 있지 않은 것이다 얻으면, 같은 일을 위해 sbt를 사용해도 좋습니다.

$ sbt clean compile stage
