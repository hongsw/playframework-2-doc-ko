독립적으로 실행 가능한 응용 프로그램 빌드

dist 작업을 사용

Play 2.0 응용 프로그램을 배포하는 가장 간단한 방법은 (git 등을 사용하여) 소스 코드를 서버에 저장하고 play start 또는 play stage 중 하나에서 응용 프로그램을 시작하는 것입니다.

그, Play 프레임 워크에 의존하지 않는 어플리케이션의 바이너리를 빌드 할 수있을 것이다. 이 경우 dist 작업에 사용할 수 있습니다.

Play 콘솔에서 dist 를 입력하자.

[ My first Application ] $ dist

conf.file 라는 특별한 시스템 속성은 외부 application.conf를 사용하여 응용 프로그램을 빌드 할 수 있습니다. 만일 운영 환경 용 application.conf 가 홈 디렉토리에 저장되어 있다고하면, 다음 명령 디폴트 이외의 application.conf 을 통합 play 응용 프로그램의 바이너리를 빌드 할 수 있습니다. bash $ play-Dconfig.file = / home / peter / prod / application.conf dist

작업이 완료되면, target 폴더에 응용 프로그램을 실행하는 데 필요한 모든 JAR 파일이 포함 된 ZIP 파일이 생성됩니다. 이 ZIP 파일의 내용은 다음과 같이 구성되어 있습니다.

my - first - application - 1.0 
 └ lib
     └  *. JAR
  └ start

응용 프로그램을 실행하려면 start 스크립트를 실행합니다.

Play 콘솔을 사용하는 대신에, OS 쉘 프롬프트에서 play dist 를 직접 시작해도 같은 것이 있습니다.

$ play dist

Maven (또는 Ivy) 리포지토리에 게시

응용 프로그램을 Maven 저장소에 게시 할 수 있습니다. 게시되는 응용 프로그램의 JAR와 POM 파일의 두 가지입니다.

게시하려면이를위한 설정을 project / Build.scala 에 기술 할 필요가 있습니다.

val main =  PlayProject ( appName , appVersion , appDependencies ). settings (
  
  publishTo : =  Some ( 
    "My resolver" at "http://mycompany.com/repo" 
  )
  
  credentials + =  Credentials ( 
    "Repo" ,  "http://mycompany.com/repo" ,  "admin" ,  "admin123" 
  )
  
)

설정을 설명하면, Play 콘솔에서 publish 작업을 수행합니다.

[ My first Application ] $ publish

위의 설정 중에 등장하는 resolver과 credentials의 상세한 정의 방법은 sbt 문서를 참조하십시오.
