새 응용 프로그램을 만들

play 명령으로 응용 프로그램을 새로 만들

새로운 응용 프로그램을 만드는 가장 기본적인 방법은 play new 명령을 사용하는 것입니다.

$ play New myFirstApp

이 명령을 실행하면 다음과 같은 필요 사항의 입력이 요구됩니다.

  • 응용 프로그램 이름 (응용 프로그램 로그 등으로 사용됩니다)
  • 사용하는 응용 프로그램의 편지지. Scala 기반, Java 기반 빈 응용 프로그램의 세 가지 중 하나를 선택할 수 있습니다.

여기에서 편지지를 선택했다고 나중에 언어를 변경할 수 없게되는 것은 아닙니다. 예를 들어 먼저 Java의 템플릿에서 응용 프로그램을 만들고 언제든지 Scala 코드를 추가 할 수 있습니다.

응용 프로그램이 생성되면 다시 play 명령을 실행하여 Play 2.0 콘솔 을 시작합시다.

$ cd myFirstApp
$ play

Play를 설치하지 않고 새 응용 프로그램을 만들

sbt를 사용하여 Play를 설치하지 않고 새 Play 응용 프로그램을 만들 수 있습니다.

필요한 sbt 0.11.2　을 설치 둡시다.

먼저 응용 프로그램을 저장하기위한 디렉토리를 만듭니다. 그 때, sbt 빌드 스크립트를 작성하여 Play를위한 설정을 2 개 추가합니다.

구체적으로는 project / plugins.sbt 에 다음 내용을 설명합니다.

/ / The typesafe repository 
resolvers + =  "Typesafe repository" at "http://repo.typesafe.com/typesafe/releases/"

/ / Use the Play SBT plugin for Play projects 
addSbtPlugin ( "play"  %  "sbt-plugin"  %  "2.0" )

사용하려는 정확한 버전이 2.0 으로 바꾸십시오. 스냅 샷 버전을 사용하려면 다음 추가 확인자 지정해야합니다.

/ / typesafe snapshots 
resolvers + =  "Typesafe Snapshots" at "http://repo.typesafe.com/typesafe/snapshots/"

또한 project / Build.scala 에 다음 내용을 설명합니다.

import SBT . _
 import  Keys . _
 import  PlayProject . _
 
object  ApplicationBuild  extends  Build  {
 
  val appName          =  "My first application" 
  val appVersion       =  "1.0"
 
  val appDependencies =  Nil
 
  val main =  PlayProject ( 
    appName , appVersion , appDependencies , mainLang = SCALA
   ) 
 
}

이러한 단계를 완료하면이 디렉토리에서 sbt 콘솔을 시작할 수 있습니다.

$ cd myFirstApp
$ sbt

sbt 콘솔을 시작하면 프로젝트가로드되어 모든 종속 모듈이 다운로드됩니다.
